# 招聘面试管理系统 - 部署指南

## 第一步：在 Supabase 创建数据库

1. 打开 https://supabase.com/dashboard
2. 创建新项目或进入已有项目
3. 点击左侧 **SQL Editor**
4. 复制 `supabase-schema.sql` 文件中的内容粘贴进去
5. 点击 **Run** 执行

> 注意：SQL 中包含 8 张表和对应的 RLS 策略，需一次性执行。

## 第二步：获取 Supabase 配置

1. 在 Supabase 控制台，点击左侧 **Project Settings**（齿轮图标）
2. 点击 **API**
3. 复制以下两个值：
   - **Project URL**: 如 `https://xxxxx.supabase.co`
   - **anon public key**: 一长串字符

## 第三步：修改配置文件

打开 `js/supabase-config.js`，将第一行的配置替换为您的实际值：

```javascript
const SUPABASE_URL = '您的Project URL';
const SUPABASE_ANON_KEY = '您的anon key';
```

> 同时更新 `dist/js/supabase-config.js` 中的相同配置。

## 第四步：部署到云端

### 方案A：EdgeOne Pages（推荐，国内访问快）

1. 确保代码已推送到 GitHub
2. 登录 [EdgeOne Pages](https://edgeone.ai)
3. 关联 GitHub 仓库
4. 设置部署目录为项目根目录
5. 自动构建部署

### 方案B：Netlify

1. 访问 https://netlify.com
2. 用 GitHub 登录
3. 选择 "Add new site" → "Import an existing project"
4. 选择 GitHub 仓库
5. 构建命令留空，发布目录设为根目录
6. 部署即可

### 方案C：Vercel

1. 访问 https://vercel.com
2. 导入 GitHub 仓库
3. Framework Preset 选 "Other"
4. 部署

### 方案D：手动部署（任何静态服务器）

将项目所有文件上传到服务器即可，无需构建步骤。

## 第五步：同步 dist 目录

如果使用 dist 目录部署，需确保 dist 目录与源文件同步：

```powershell
# 同步 HTML 文件
copy index.html dist/
copy interview.html dist/

# 同步 JS 文件
copy js\supabase-config.js dist\js\
copy js\admin.js dist\js\

# 同步 CSS 文件
copy css\admin.css dist\css\
```

## 访问地址

- 管理后台: `部署网址/index.html`
- 候选人端: `部署网址/interview.html?link=面试链接`

## 功能说明

### HR 管理端
1. 在「招聘类别」中创建大类（社会招聘/校园招聘）
2. 在「岗位管理」中添加岗位，配置评分维度
3. 在「题库管理」中选择岗位，添加或批量导入面试题目
4. 在「面试邀请」中创建邀请，复制面试链接发给候选人
5. 候选人完成面试后，在「面试记录」中查看详情和导出

### 候选人端
1. 通过面试链接访问，填写个人信息
2. 开启摄像头和麦克风
3. AI 面试官语音提问，候选人用语音或键盘回答
4. 面试完成后可导出记录和视频

## 注意事项

- 语音识别功能需要 Chrome 浏览器，且需在 HTTPS 环境下运行
- 摄像头和麦克风需要用户授权
- Supabase 免费版有 500MB 存储和 50K 月活请求限制
- 建议为生产环境配置 Supabase RLS 策略（当前为公开读写）
